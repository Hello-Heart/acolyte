Acolyte Get a tour

You want to know how Acolyte is useful to test JDBC access? Here is a tour about.

Usual way to setup an Acolyte connection can be summed up with following steps.

  1. You indicate which access case you want to handle, which statement you want to handle as query or update. It's not required exactly simulate DB behaviour, just data exchanges abstracted by JDBC that are required for testing.
  2. For each case you describe what is the result (either update count or result set).
  3. Then, Acolyte connection can be passed/referenced for testing/mockup anywhere JDBC one is required to access.
