You want to know how Acolyte is useful to test ReactiveMongo persistence?
Here is a tour about.

Usual way to setup an Acolyte for ReactiveMongo can be summed up with following steps.

  1. You indicate which access case you want to handle, which command you want to handle as query or write operations. It's not required to exactly simulate MongoDB behaviour, only data exchanges abstracted by the driver are required for testing.
  2. For each case you describe what is the response (either update count, result set or DB error).
  3. Then, Acolyte instances of ReactiveMongo driver or connection can be used for testing/mockup.
